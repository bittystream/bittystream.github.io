具有较好的内存管理意识

Bitty的博客
2021-03-09

好久没写博客了，记录一下子。

这学期选了多媒体课，做作业会用到Pr，没想到打开一个项目把素材拖到时间轴的时候居然直接要给Adobe发送崩溃报告？查看报错信息，显示的是EXCEPTION_IN_PAGE_ERROR。在win32文档里查到了关于这个异常的定义，大概就是说线程访问了无法加载的页，这个异常是在读写映射文件而不是页文件的时候导致的。

The thread tried to access a page that was not present, and the system was unable to load the page. For example, this exception might occur if a network connection is lost while running a program over the network.

Pr报这个异常后，蓝屏接踵而至，报错为KERNEL_DATA_INPAGE_ERROR。有可能是虚拟内存页面文件中存在坏簇、病毒、磁盘控制其错误、内存有问题。从页面文件读/加载存储在硬盘驱动器的信息存在一些问题。

反复卸载安装各个版本的Pr，没有结果。用图形界面里的磁盘碎片整理工具挨了一下午，没有结果。开始在电商平台和问答网站物色新电脑，但作为一个边缘人，三问何去何从，竟难以从各色笔记本中找到真正适合自己在未来长期使用的那一款。深夜深思，辗转反侧：多媒体技术，你就是我的蚁穴？

是日下午再次搜索INPAGE ERROR，发现用chkdsk命令来修复磁盘也是一种办法。一个多小时过去，第四个阶段的修复已完毕，电脑自动重启。惊坐起，打开Pr。很卡很慢，心想，也不能全怪这磁盘修复，毕竟刚开机的时候都不快。随后不抱任何希望地随便点开一个名为test_again的项目，它之前还有test、test_1、test_2，它们都为我带来了崩溃报告；不抱任何希望地随便点开一个文件夹；不抱任何希望地随便导入一个视频，然后把它拖到进度条上。

没有崩溃报告，视频正常导入播放，可以正常地编辑。尖叫几声，奔走相告，然后写下了这些无聊的文字。

如果chkdsk /r没有用，或许我会下载更古早版本的Pr。或者我根本不会用Pr。或许我会重装系统。反正，换电脑是没那么斩钉截铁的，永远都不会。
